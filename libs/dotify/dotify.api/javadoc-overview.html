Provides interfaces for embossing, creating and manipulating PEF-files. The API supports both SPI and OSGi runtime contexts.

Providing implementations

The following interfaces have been constructed so that they can be discovered as SPI and OSGi services:

  • org.daisy.dotify.api.embosser.EmbosserProvider
  • org.daisy.dotify.api.embosser.EmbosserCatalogService
  • org.daisy.dotify.api.table.TableProvider
  • org.daisy.dotify.api.table.TableCatalogService
  • org.daisy.dotify.api.paper.PaperProvider
  • org.daisy.dotify.api.paper.PaperCatalogService
  • org.daisy.dotify.api.validator.ValidatorProvider
  • org.daisy.dotify.api.validator.ValidatorFactoryService

While all of the above may be discovered by SPI or OSGi, the following interfaces provide a starting point for a developer who wishes to add support for additional embossers:

  • org.daisy.dotify.api.embosser.EmbosserProvider
  • org.daisy.dotify.api.table.TableProvider
  • org.daisy.dotify.api.paper.PaperProvider

Implementations bundled with the API

Default implementations of the following interfaces are included:

  • org.daisy.dotify.api.embosser.EmbosserCatalogService
  • org.daisy.dotify.api.paper.PaperCatalogService
  • org.daisy.dotify.api.table.TableCatalogService
  • org.daisy.dotify.api.validator.ValidatorFactoryService

In an OSGi context, an implementation is retrieved from the service registry using these interfaces.

In an SPI context, the implementations can be accessed directly:

  • org.daisy.dotify.api.embosser.EmbosserCatalog
  • org.daisy.dotify.api.paper.PaperCatalog
  • org.daisy.dotify.api.table.TableCatalog
  • org.daisy.dotify.api.validator.ValidatorFactory
