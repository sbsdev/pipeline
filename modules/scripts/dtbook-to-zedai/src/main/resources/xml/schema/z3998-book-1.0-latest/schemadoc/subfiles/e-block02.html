The toc block element is used to group any ancillary information that falls under an entry's leader line (not including subentries), such as descriptions or topic lists.

Allowed parents

entry

Allowed children

This element must not contain text.

This element may contain the following children: aside, block, hd, list, object, p and transition

Content model and additional requirements

optional attributes: @about, @by, @class, @content, @continuation, @datatype, @desc, @its:dir, @its:translate, @property, @rel, @resource, @rev, @role, @sel:selid, @ssml:onlangfailure, @typeof, @xml:base, @xml:id, @xml:lang and @xml:space
one or more of
any of the 7 elements: aside, block, hd, list, object, p and transition
Note that in addition to restrictions presented in the content model above, use of this element must also respect the following requirement:
  • The block element must neither be empty nor contain only whitespace.
Such requirements take precedence over any conflicting statements in the content model or in the lists above of allowed children and parents.

Variants

This element exists in 2 variants. The other variant is the block variant.
